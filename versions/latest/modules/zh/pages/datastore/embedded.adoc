= 嵌入式数据存储
:page-languages: [en, zh]
:revdate: 2025-08-08
:page-revdate: {revdate}

使用嵌入式数据存储意味着利用在Kubernetes集群内运行的数据库，通常作为容器化服务，例如etcd。此选项简化了部署，并可能提高性能和安全性。替代方案是xref:datastore/external.adoc[外部数据库]。

== 数据存储选项

[WARNING]
.实验性
====
RKE2官方支持嵌入式etcd，嵌入式SQLite被视为实验性功能。
====

* *嵌入式 https://etcd.io/[etcd]*:嵌入式etcd是默认的数据存储，如果没有其他数据存储配置，将使用它。
* *嵌入式 https://www.sqlite.org/index.html[SQLite]*:SQLite不能在多个服务器的集群上使用。它使用项目 https://github.com/k3s-io/kine[kine]。

== 嵌入式etcd

嵌入式 https://etcd.io/[etcd]是默认的数据存储，如果没有其他数据存储配置，将使用它。 这是唯一允许在xref:install/ha.adoc[HA模式]下部署RKE2的嵌入式选项。除非明确取消设置，否则每个RKE2服务器将部署一个etcd pod，所有etcd实例将保持法定人数。RKE2包含工具，可以在使用此数据存储时轻松创建快照，如xref:datastore/backup_restore.adoc[备份和恢复]说明中所述。

== 嵌入式SQLite

此嵌入式选项不推荐用于生产，但如果您需要运行一个简单的、短暂的集群，例如在CI/CD环境中，它可能会很有用。在使用 https://www.sqlite.org/index.html[SQLite]进行部署时，不支持HA模式。

=== 使用SQLite的单服务器

. 在配置文件中设置`disable-etcd`而不使用`server`参数。
+
[,yaml]
----
disable-etcd: true
----

. 安装RKE2。
+
[,bash]
----
curl -sfL https://get.rke2.io | sh -
----

. 启用 rke2-server 服务。
+
[,sh]
----
systemctl enable rke2-server.service
----

. 启动 rke2-server 服务。
+
[,sh]
----
systemctl start rke2-server.service
----

您可以通过 `kubectl get nodes` 跟踪服务器的启动过程，以查看服务器获得 `Ready` 状态。有关如何访问 RKE2 的更多信息，请参见 xref:cluster_access.adoc[集群访问]。
