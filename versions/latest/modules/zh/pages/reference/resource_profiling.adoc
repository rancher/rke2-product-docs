本节记录了确定 RKE2 最低资源需求的测试结果。

== 资源测试范围

资源测试旨在解决以下问题：

* 在单节点集群上，确定整个 RKE2 服务器堆栈的合法最低处理器和内存量，假设将在集群上部署实际工作负载。
* 在代理节点上，确定应为 kubelet 和 RKE2 代理组件预留的合法最低处理器和内存量。

=== 环境和组件

|===
| 架构 | OS | 系统 | CPU | RAM | 磁盘

| x86_64
| Ubuntu 22.04
| AWS c6id.xlarge
| 英特尔至强铂金8375C CPU，4核 2.90 GHz
| 8 GB
| NVME SSD
|===

测试的组件包括：

* RKE2 v1.27.12，启用所有软件包组件，使用 Canal 作为 CNI。
* https://kubernetes.io/docs/tasks/run-application/run-stateless-application-deployment/[Kubernetes 示例 Nginx 部署]

=== 方法

`systemd-cgtop` 用于跟踪 systemd cgroup 级别的处理器和内存利用率。

* `system.slice/rke2-server.service` 跟踪 RKE2 和 containerd 组件的资源利用率。
* `system.slice/rke2-agent.service` 跟踪代理组件的资源利用率。

利用率数据基于在运行所述工作负载的节点上进行的稳态操作的第95百分位数读数，从而给出典型资源使用的上限。
