== WindowsでCalicoのBGPを有効にする

LinuxとWindowsノードの両方を使用してRKE2でトラフィックをルーティングするためにBGPモードを有効にするには、サーバー設定でCalico CNIを選択する必要があります。

[,yaml]
----
# /etc/rancher/rke2/config.yaml
cni: calico
----

次に、BGPを有効にし、カプセル化を`None`に設定してCalicoを構成する必要があります。

[,yaml]
----
# /var/lib/rancher/rke2/server/manifests/rke2-calico-config.yaml
apiVersion: helm.cattle.io/v1
kind: HelmChartConfig
metadata:
  name: rke2-calico
  namespace: kube-system
spec:
  valuesContent: |-
    installation:
      calicoNetwork:
        bgp: Enabled
      backend: "None"
----

== Windowsノードを起動しています

xref:install/quickstart.adoc#_windows_agent_worker_node_installation[こちら]に記載されているように、WindowsノードでRKE2を構成します。

RKE2サービスを開始する前に、`RemoteAccess`を有効にする必要があります。

[,powershell]
----
Install-WindowsFeature RemoteAccess
Install-WindowsFeature RSAT-RemoteAccess-PowerShell
Install-WindowsFeature Routing
----

これには、正常に機能するために再起動が必要です。

次に、`RemoteAccess`サービスを設定します：

[,powershell]
----
Install-RemoteAccess -VpnType RoutingOnly
----

`RemoteAccess`サービスが正常に開始されたかどうかを確認するには、次のコマンドを使用します：

[,powershell]
----
Start-Service RemoteAccess
----

その後、RKE2サービスを開始できます。
