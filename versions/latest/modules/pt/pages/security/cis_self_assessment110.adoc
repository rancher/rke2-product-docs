== Visão Geral

Este documento é um complemento ao guia de proteção do RKE2. O guia de proteção fornece orientações prescritivas para a proteção de uma instalação de produção do RKE2, e este guia de referência é destinado a ajudá-lo a avaliar o nível de segurança do cluster protegido em relação a cada controle no benchmark do CIS Kubernetes. Deve ser utilizado por operadores de RKE2, equipes de segurança, auditores e tomadores de decisão.

Este guia é específico para a linha de lançamento *v1.28-1.31* do RKE2 e a versão *v1.10* do Benchmark do CIS Kubernetes.

Para mais informações sobre cada controle, incluindo justificativas detalhadas e descrições de verificações, você pode consultar a seção correspondente do Benchmark do CIS Kubernetes v1.8. Você pode baixar o benchmark, após criar uma conta gratuita, no https://www.cisecurity.org/benchmark/kubernetes/[Centro para Segurança da Internet (CIS)].

=== Metodologia de testes de controles

Cada controle no Benchmark do CIS Kubernetes foi avaliado em um cluster RKE2 configurado de acordo com o guia de proteção que o acompanha.

Estes são os resultados possíveis para cada controle:

* *APROVADO* - O controle é automatizado (pontuação: verdadeiro). O cluster RKE2 em teste passou na auditoria descrita no benchmark.
* *Não Aplicável* - O controle não é aplicável (tipo: pular) ao RKE2 devido à forma como foi projetado para operar. A seção de justificativa explicará por que isso ocorre.
* *AVISO* - O controle é manual (pontuação: falso) no benchmark do CIS e depende da intervenção manual do operador. A seção de remediação fornecerá orientações sobre como alcançar um resultado APROVADO.
