== Übersicht

Dieses Dokument ist ein Begleitdokument zum RKE2-Sicherheits-Härtungsleitfaden. Der Härtungsleitfaden bietet verbindliche Anleitungen zur Härtung einer Produktionsinstallation von RKE2, und dieser Benchmark-Leitfaden soll Ihnen helfen, das Sicherheitsniveau des gehärteten Clusters im Vergleich zu jeder Kontrolle im CIS Kubernetes Benchmark zu bewerten. Er ist für RKE2-Betreiber, Sicherheitsteams, Prüfer und Entscheidungsträger gedacht.

Dieser Leitfaden ist spezifisch für die *v1.28-1.31* Release-Linie von RKE2 und die *v1.10* Version des CIS Kubernetes Benchmark.

Für weitere Informationen zu jeder Kontrolle, einschließlich detaillierter Begründungen und Beschreibungen der Überprüfungen, können Sie auf den entsprechenden Abschnitt des CIS Kubernetes Benchmark v1.8 verweisen. Sie können den Benchmark herunterladen, nachdem Sie ein kostenloses Konto erstellt haben, im https://www.cisecurity.org/benchmark/kubernetes/[Center for Internet Security (CIS)].

=== Testmethodik für Kontrollen

Jede Kontrolle im CIS Kubernetes Benchmark wurde gegen ein RKE2-Cluster bewertet, das gemäß dem begleitenden Härtungsleitfaden konfiguriert wurde.

Dies sind die möglichen Ergebnisse für jede Kontrolle:

* *BESTANDEN* - Die Kontrolle ist automatisiert (bewertet: wahr). Das getestete RKE2-Cluster hat die im Benchmark skizzierte Prüfung bestanden.
* *Nicht anwendbar* - Die Kontrolle ist nicht anwendbar (Typ: überspringen) für RKE2, aufgrund der Art und Weise, wie sie konzipiert ist zu arbeiten. Der Abschnitt zur Begründung wird erklären, warum dies so ist.
* *WARNUNG* - Die Kontrolle ist manuell (bewertet: falsch) im CIS-Benchmark und hängt von der manuellen Intervention des Betreibers ab. Der Abschnitt zur Behebung wird Anleitungen geben, wie ein BESTANDEN-Ergebnis erzielt werden kann.
